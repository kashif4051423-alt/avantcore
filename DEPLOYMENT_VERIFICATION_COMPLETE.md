## Technical Details

### Why Nested Structure Required vercel.json
The repository has this structure:
```
Root (package-lock.json, .git, README.md)
└── AvantCore-Technology-website-main/ (Actual Vite project)
    ├── package.json
    ├── vite.config.ts
    └── src/, public/, etc.
```

Vercel's default behavior:
- Looks for package.json at repository root ❌
- Would try to build from root
- Wouldn't find the actual project

Solution - vercel.json:
- Tells Vercel exact location of package.json ✓
- Specifies build command with correct path ✓
- Specifies output directory with correct path ✓

### Build Output Explanation
```
Generated Bundles:
- vendor.js (20.20 kB): React, React-DOM, React Router
- animations.js (137.04 kB): Framer Motion library
- ui.js (187.14 kB): Shadcn UI components
- index.js (495.96 kB): Application code
- index.css (124.13 kB): Tailwind CSS + Components

Total: 752.27 kB (optimized and minified)
```

### SPA Routing
- index.html serves all routes
- React Router handles client-side navigation
- No server-side routing needed
- Perfect for Vercel static hosting
