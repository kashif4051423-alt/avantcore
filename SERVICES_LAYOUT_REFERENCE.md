## Animation Patterns

### Pattern 1: Fade + Y-Translate
Used in most sections
```javascript
initial={{ opacity: 0, y: 20 }}
whileInView={{ opacity: 1, y: 0 }}
transition={{ delay: idx * 0.1 }}
```

### Pattern 2: Scale
Used for grid items
```javascript
initial={{ opacity: 0, scale: 0.95 }}
whileInView={{ opacity: 1, scale: 1 }}
```

### Pattern 3: X-Translate (Alternating)
Used for timeline/process items
```javascript
initial={{ opacity: 0, x: idx % 2 === 0 ? -40 : 40 }}
whileInView={{ opacity: 1, x: 0 }}
```

### Pattern 4: 3D Rotate
Used for governance pillars
```javascript
initial={{ opacity: 0, rotateY: -20 }}
whileInView={{ opacity: 1, rotateY: 0 }}
```

### Pattern 5: Image Zoom
Used on hover
```javascript
whileHover={{ scale: 1.03 }}
transition={{ duration: 0.5 }}
```

## Responsive Grid Patterns

### 1-Column (Mobile)
All layouts stack to single column on small screens

### 2-Column (Tablet)
- Features: `grid-cols-1 md:grid-cols-2`
- Timeline: Maintains stacked appearance
- Tiers: Stacks to single for comparison

### 3-Column (Desktop)
- Benefits: `lg:grid-cols-3`
- Features: `md:grid-cols-2 lg:grid-cols-3`
- Analytics: `md:grid-cols-2`

## Customization Guide

To modify a layout:

1. Find service in `serviceDetails` object
2. Update `layout` field to match component name
3. Add service-specific data (features, items, etc.)
4. Create custom layout component
5. Add to render logic in `ServiceDetail` component

Example:
```javascript
'new-service': {
  title: 'New Service',
  layout: 'newlayout',
  customData: [...],
}

// Then create:
const NewlayoutLayout = ({ service }: any) => { ... }

// Add to render:
{layout === 'newlayout' && <NewlayoutLayout service={service} />}
```
